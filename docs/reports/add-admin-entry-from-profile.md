# Добавление входа в админку с экрана профиля

## Исходная UX-проблема

У пользователей с ролью `admin` в storefront UI не было очевидной точки входа в админ-панель. Роль уже отображалась на экране `Профиль`, но из этого состояния нельзя было быстро и понятно перейти в `/admin`, из-за чего админ-функции были скрыты за неявным маршрутом.

## Внесённые изменения

- На экране `Профиль` добавлена отдельная заметная карточка входа в админ-панель.
- CTA показывается только для пользователей, у которых текущий профиль проходит проверку `hasAdminRole(profile.role)`.
- Для обычных пользователей CTA не рендерится вообще.
- Проверка видимости CTA переиспользует тот же предикат роли, что и server-side guard админ-страниц и admin API.
- Подпись роли на профиле приведена к русскому виду: `Администратор` или `Покупатель`.
- Для новой карточки добавлены storefront-стили с мобильной адаптацией и `focus-visible` состоянием, без переработки admin internals.
