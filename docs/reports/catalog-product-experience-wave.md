# Catalog + Product Experience Wave

## Исходные проблемы
- На тёмных поверхностях нужно было жёстче приоритизировать читаемость и не допускать слабого контраста текста.
- Экран каталога оставался перегружен для top-level сценария и не выглядел как чистая витрина категорий.
- Каталогу не хватало визуального наполнения: без достаточного числа категорий плиточная подача выглядела пустой.
- Карточки товаров были мягче предыдущей волны, но всё ещё уступали shopping-референсу по торговой иерархии.
- На странице товара не хватало понятной основы под размеры, варианты, характеристики и отзывы.

## Выбранный подход
- Делать storefront-first изменения без risky refactor схемы данных, корзины, import flow и admin flows.
- Использовать presentation-layer расширения поверх существующей модели товара: новые UI-секции не ломают cart logic и совместимы с текущими admin/DB данными.
- Добавить безопасочное витринное наполнение каталога через presentation categories: реальные категории остаются приоритетными, недостающие добираются демо-слоем.

## Внесённые изменения
- Усилен shopping UX каталога: top-level каталог сосредоточен на плитках категорий, а фильтры, подборки, баннер и product results показываются только в углублённом режиме.
- В data layer добавлено безопасное заполнение до 20 категорий через `presentationCategoryPool`, с приоритетом реальных данных и fallback только для витринного представления.
- Товарные карточки усилены под marketplace-подачу: крупная медиазона, заметная цена и старая цена, discount badge, favorite overlay и отдельный CTA в корзину прямо в листинге.
- На странице товара добавлена новая поверхность `StoreProductExperience` с выбором цвета/размера/модификации, размерной сеткой, превью отзывов и отдельной секцией характеристик с modal surface.
- Product presentation вынесен в отдельный builder, который безопасно достраивает option groups, size guide, specification groups и review previews без изменения существующей product schema в БД.
- Новый русский UI приведён к более чистым формулировкам, без англоязычных helper-текстов и без технических слов в пользовательской подаче.

- `pnpm run lint` — пройдено успешно.

- В обоих прогонах остаются только информационные предупреждения `baseline-browser-mapping`.

## Остаточные ограничения / follow-ups
- Выбор варианта на странице товара пока presentation-only: корзина по-прежнему добавляет базовую карточку товара без отдельной variant-модели.
- Блок отзывов пока intentionally demo-safe: UI готов к будущему backend, но реальные отзывы и рейтинги не подключались.
- 20 категорий в каталоге реализованы как безопасное витринное заполнение; при появлении полного реального набора категорий этот слой должен постепенно перестать быть заметным.
- Реальную оплату, provider-specific integration, import engine и Excel support matrix `XLSX/XLS/XLSM/XLTX` в рамках волны не меняли.
